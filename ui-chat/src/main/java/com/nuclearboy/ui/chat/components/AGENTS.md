## 目录职责

`components` 是 `ui-chat` 的组件层，组合零件和 Compose 基础控件，提供可复用的聊天 UI 功能单元，例如命令按钮、输入辅助控件、带脱敏诊断指纹和正式链路测试口径的正式聊天失败诊断卡、工具型草稿预警条、带工具协议诊断和正式链路口径的工具受限提示卡、后置结果复核警告卡、通用警告卡底座、文件面板过滤输入、过滤输入命中数量持续显示、文件面板空结果状态、带数值语义的文件面板摘要条、带排序语义的文件面板排序切换条，以及支持全选可见、全选/补选数量标签、取消可见、取消匹配、取消隐藏、图标操作数量语义、只看/全部切换数量标签、当前可见直接引用、只看已选、引用匹配、引用按钮数量标签、已选大小展示、可见选择进度、隐藏已选提示和选中态两行布局的文件选择操作条。

## 边界

组件层可以渲染局部 UI 并向上抛出用户意图，例如工具型草稿预警里的“追加防假执行提示”，但不直接访问 ViewModel、Agent、文件系统、网络或导航。

## 允许依赖

允许依赖 `ui-chat/parts`、Compose UI、Material 3 和聊天主题。

## 禁止事项

不要在组件层保存业务状态、执行斜杠命令、修改聊天记录或输出明文秘密值。

## 常用命令

- `./gradlew :ui-chat:compileDebugKotlin`
- `./gradlew :app:assembleDebug`

## 验证方式

验证组件在空对话、有历史消息和处理中三种状态下的显示与点击回调；正式聊天失败诊断卡、工具受限提示卡和后置结果复核警告卡要验证语义描述、标题、摘要、诊断指纹、正式链路测试口径和下一步不换行溢出，其中工具受限卡必须显示 `tool.protocol`；工具型草稿预警条要验证语义描述、标题、摘要、追加防假执行按钮和下一步不换行溢出；文件面板组件还要验证空查询、输入过滤、过滤输入命中数量持续显示、无匹配清除过滤入口、清除回调、摘要条横向滚动、摘要条数值语义、排序切换状态、排序按钮语义、全选可见、全选/补选数量标签、取消可见、取消匹配、取消隐藏、图标操作数量语义、只看/全部切换数量标签、当前可见直接引用、只看已选、引用匹配、引用按钮数量标签、已选大小展示、可见选择进度、隐藏已选提示、选中态两行布局和已选操作条回调。
