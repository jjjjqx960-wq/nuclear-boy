## 目录职责

这里承载 `ui-chat` 零件层的 JVM 单元测试，重点覆盖文件面板过滤、当前显示列表规则、过滤摘要、排序、摘要、选择、全选可见、取消可见、隐藏选择清理、只看已选、已选大小汇总、选择操作条显示条件、选择状态文案、隐藏已选提示、引用提示和聊天输入辅助等纯函数。

## 边界

测试应只验证 `parts` 层稳定规则，不启动 Compose、不访问 ViewModel、不读写真实文件系统。

## 允许依赖

允许依赖 `ui-chat/src/main/.../parts`、`common` 模型、JUnit 和 Kotlin 标准库。

## 禁止事项

不要在测试中复述或构造真实账号、Token、密钥和个人数据。不要把 UI 布局断言写入零件层测试。

## 常用命令

- `./gradlew :ui-chat:testDebugUnitTest`

## 验证方式

用小型内存数据覆盖排序优先级、大小写、空扩展名、目录优先、数量统计、文件面板过滤、当前显示列表规则、过滤摘要、选择切换、全选可见、取消可见、隐藏选择清理、只看已选、已选大小汇总、选择操作条显示条件、选择状态文案、隐藏已选提示和提示格式等规则。
