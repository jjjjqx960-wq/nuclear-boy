## 目录职责

`app/ui/settings` 承载设置页整体 UI、状态接线和用户触发的设置流程。

## 边界

这里可以编排设置页 ViewModel、表单、输入期提示、服务地址和模型名清理提示、端点预览、完整地址风险提示、风险恢复动作、完整地址补全动作、模型列表获取按钮、模型列表点选区、模型列表清空筛选动作、测试按钮、剪贴板动作、请求上下文接线、失败操作建议接线和结果展示，但可复用的展示文案、状态判断、Key 脱敏指纹、请求上下文摘要、模型列表摘要、模型列表筛选提示、模型列表清空筛选动作文案、失败操作建议、复制摘要和格式化规则应下沉到 `parts`。

## 允许依赖

可以依赖 `app` 装配能力、`api-deepseek` 的模型配置接口、`common` 的结果类型，以及同目录 `parts` 零件。

## 禁止事项

不要在这里实现模型协议细节、网络请求拼接、密钥明文输出或跨模块业务逻辑。

## 常用命令

- `./gradlew :app:compileDebugKotlin`
- `./gradlew :app:testDebugUnitTest`
- `./gradlew :app:assembleDebug`

## 验证方式

修改设置页后优先验证编译、相关单元测试、Key 指纹脱敏展示、输入期服务地址和模型名隐藏字符清理提示、端点预览、完整地址风险提示、风险恢复动作、完整地址补全动作、模型列表获取/过滤/清空筛选/点选填入、请求上下文摘要、失败操作建议、模型测试/全量自检复制摘要脱敏和 Debug APK 打包。
