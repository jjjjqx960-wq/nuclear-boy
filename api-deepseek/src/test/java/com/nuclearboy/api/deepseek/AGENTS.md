## 目录职责

这里承载 `api-deepseek` 组件层的本地 JVM 单元测试，覆盖 DTO、协议解析、endpoint 构造、诊断提示、模型名和服务地址清理规则。

## 边界

只测试纯逻辑和本地序列化行为，不访问真实网络、不依赖 Android 设备、不读取本机私有配置。

## 允许依赖

可以依赖 `api-deepseek` 源码、`common` 类型、JUnit 和项目已有测试依赖。

## 禁止事项

不要在测试中写入真实 API Key、Token、账号密码、签名密钥或个人数据。

## 常用命令

- `./gradlew :api-deepseek:test`

## 验证方式

新增协议或清理规则时补充正反向用例，至少覆盖正常输入、隐藏字符输入和不会误伤普通首尾空格的场景。
