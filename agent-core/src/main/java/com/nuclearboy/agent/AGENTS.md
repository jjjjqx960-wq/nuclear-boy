## 目录职责

`agent` 包承载 Agent 部件层核心编排，包括主循环、系统提示、工具注册、工具调用保护、重试错误门控和事件模型。

## 边界

这里只实现 Agent 编排与工具抽象，不实现 Android UI、具体网络协议细节、文件系统工具实现或应用入口。

## 允许依赖

可以依赖 `api-deepseek` 的消息/工具 DTO、`common` 的错误和模型、协程与序列化基础库。

## 禁止事项

不要输出明文 Token、账号密码、授权头或完整个人数据。不要引入固定工具调用轮数上限；应使用最终回复、用户取消、上下文保护、重复工具调用保护和最终错误门控等明确条件停止。

系统提示不得鼓励模型在普通文本里伪造工具执行。任何“已读取、已写入、已测试、已安装、已验证”类结论必须来自真实工具结果或明确的外部证据。

## 常用命令

- `./gradlew :agent-core:test`
- `./gradlew :agent-core:compileDebugKotlin`

## 验证方式

修改主循环或工具调用保护后，至少运行 `:agent-core:test`；涉及集成行为时补跑 `:app:assembleDebug` 和实机聊天旅程。
